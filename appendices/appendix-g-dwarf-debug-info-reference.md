## G.5 Line Table Opcodes

The line number table encodes source-to-address mappings using a state machine.

### Standard Opcodes (DW_LNS_*)

| Opcode | Value | Description |
|---|---|---|
| `DW_LNS_copy` | 1 | Emit row; reset `basic_block`, `prologue_end`, `epilogue_begin` |
| `DW_LNS_advance_pc` + ULEB128 | 2 | Add to address; adjust `op_index` |
| `DW_LNS_advance_line` + SLEB128 | 3 | Add to `line` |
| `DW_LNS_set_file` + ULEB128 | 4 | Set `file` register |
| `DW_LNS_set_column` + ULEB128 | 5 | Set `column` register |
| `DW_LNS_negate_stmt` | 6 | Toggle `is_stmt` |
| `DW_LNS_set_basic_block` | 7 | Set `basic_block` flag |
| `DW_LNS_const_add_pc` | 8 | Advance PC by address increment for opcode 255 |
| `DW_LNS_fixed_advance_pc` + 2-byte | 9 | Add fixed value to address |
| `DW_LNS_set_prologue_end` | 10 | Set `prologue_end` flag |
| `DW_LNS_set_epilogue_begin` | 11 | Set `epilogue_begin` flag |
| `DW_LNS_set_isa` + ULEB128 | 12 | Set ISA register |

### Extended Opcodes (DW_LNE_*)

| Opcode | Encoding | Description |
|---|---|---|
| `DW_LNE_end_sequence` | 0x01 | Mark end of instruction sequence; emit row; reset state |
| `DW_LNE_set_address` + address | 0x02 | Set PC to absolute address |
| `DW_LNE_define_file` + name + ... | 0x03 | Add file entry (DWARF 4) |
| `DW_LNE_set_discriminator` + ULEB128 | 0x04 | Set discriminator (block within line) |
| `DW_LNE_lo_user` | 0x80 | User-defined opcode range start |
| `DW_LNE_hi_user` | 0xff | User-defined opcode range end |

### Special Opcodes

Opcode values 13â€“255 encode combined line/address advances. Given an opcode `op`:
- `adjusted = op - opcode_base`
- `line_increment = line_base + (adjusted % line_range)`
- `address_advance = (adjusted / line_range) * minimum_instruction_length`

## G.9 `.debug_names` (DWARF 5 Name Index)

The `.debug_names` section provides accelerated lookup of names (functions, types, variables) without scanning all DIEs. Structure:

| Component | Description |
|---|---|
| Header | Unit length, version, padding, CU count, TU count, foreign TU count, bucket count, name count, abbrev table size, augmentation string |
| CU offsets array | `.debug_info` offsets of all CUs covered by this index |
| TU offsets array | `.debug_info` or `.debug_types` offsets of covered TUs |
| Foreign TU sigs | 8-byte type signatures for foreign TUs |
| Hash buckets | Hash table (bucket array + name/entry pointers) |
| Name table | `{string_offset, entry_offset}` pairs |
| Abbreviation table | Maps abbreviation code to `(DW_TAG, IDX_*)` attribute lists |
| Entry pool | Lookup entries: `(CU_index, DIE_offset, [parent_DIE_offset])` |

Key `DW_IDX_*` attributes used in entries:

| Attribute | Description |
|---|---|
| `DW_IDX_compile_unit` | Index into CU offsets array |
| `DW_IDX_type_unit` | Index into TU offsets array |
| `DW_IDX_die_offset` | Offset within the CU's `.debug_info` |
| `DW_IDX_parent` | Entry pool offset of parent entry |
| `DW_IDX_type_hash` | 8-byte type signature |

## G.10 Compact Unwind (macOS Apple Format)

macOS uses its own compact unwind format in `__TEXT/__unwind_info` alongside (or instead of) `.eh_frame`.

| Encoding bits [31:28] | Mode | Description |
|---|---|---|
| 0 | Frameless | Uses RBP or RSP-based frame; encoded in bits 8:0 |
| 1 | DWARF | Uses `.eh_frame` CIE/FDE (fallback) |
| 2 | RBP frame | Saves: RBP at [rbp], then N registers at [rbp-8N,...] |
| 3 | Stack-immediate | Frameless; stack size in encoding; saved regs listed |
| 4 | Stack-indirect | Frameless; stack size from memory location |

Register encoding in compact unwind (x86-64): 3-bit per register: `0`=unused, `1â€“6`= `rbx/r12/r13/r14/r15/rbp`.
